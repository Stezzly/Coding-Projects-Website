Welcome to XML!

This guide provides an easy-to-follow introduction to XML, aimed at those who are new to coding. XML, or Extensible Markup Language, is a versatile markup language used for storing and transporting data.
By the end of this guide, you will understand the basics of XML, including how to create XML documents and how to structure data effectively within them.
Let's dive into the world of XML and explore how it is used to format information in a structured, readable way.

What You'll Need

To begin with XML coding, you don't need complex software or systems. All you need is:

  • A simple text editor like Notepad++, Visual Studio Code, or any XML-friendly editor.
  • Basic knowledge of HTML can be helpful since XML uses a similar structure.

XML Basics

In this section, we'll cover the foundational elements of XML:

  • Understanding XML Syntax and Rules.
  • Creating your first XML document.
  • Learning about elements, tags, attributes, and nesting.

Practical Example

We'll also go through a practical example where you'll create a simple XML file to store data about books in a library.

The first step is to select a text editor. XML is a markup language, not a programming language, so it does not require a program to complile the codes. We can use something as simple as any built in text editor on our device, although that is usually not the most preferable method.

Notepad++ or Visual Studio Code are two recommendations suggested for using XML

  1. Visual Studio Code for maintaining different languages and code all in the one same location - Ease of access
  2. Notepad++ for text simplicity as well as minimal features: Easy to not get sidetracked

Begin by opening your chosen XML-friendly text editor. Create a new file and start with the XML declaration, which specifies the XML version and the encoding used. A typical XML file looks like this:

        <?xml version="1.0" encoding="UTF-8"?>
        <books>
            <book>
                <title>Example Book Title</title>
                <author>Author Name</author>
            </book>
        </books>

Save your file with an .xml extension. Try adding different elements and attributes to familiarize yourself with the structure of XML.


Validating your XML file is crucial to ensure it is well-formed and adheres to the XML standards. Use an online validator like XMLValidation.com to check your XML structure:

Copy and paste your XML code into the validator. It will check for syntax errors and ensure that your tags are properly closed and nested. Understanding how to debug and correct errors in your XML files is an essential skill for working with XML.

Congratulations! You have taken your first steps into XML coding.
